Concurrency and Transactions

The default number of concurrent connections per route for Android is two, with 10 maximum total connections. Sending more than the maximum connections per route will block on the additional requests until the earlier requests complete.

The Buddy SDK uses up to two connections per Buddy route. The app dev is free to modify the maximum total connections limit, by using the overloaded constructor in the AsyncHttpClient, but the SDK will not make use of a higher per‑route connection limit, as per the HTTP protocol definition (http://www.w3.org/Protocols/rfc2616/rfc2616-sec8.html). The SDK manages any multi-part web requests within the SDK (for example, uploading a single photo). The app dev is responsible for any multi-part requests wrapping the SDK (e.g. adding multiple photos to an album).

The SDK will not retry failed requests. All failures will be reported as part of the response callback.

Configuration Management

The SDK allows the app dev to configure the following transport-level options:

·        setupHttpClient(int socketTimeoutValue, int maxConnections, int socketBufferSize)

o   socketTimeoutValue - Default socket timeout (SO_TIMEOUT in the Android SDK) in milliseconds, which is the timeout to wait for data. A timeout value of zero is interpreted as an infinite timeout. Value of -1 sets the default value of 10 seconds.

o   maxConnections - Maximum number of connections allowed. Value of -1 sets the default value of 10.

o   socketBufferSize - Socket buffer size in bytes. Value of -1 sets the default value of 8192.

Exception Management

Generically, exceptions or errors within the Buddy SDK will not be thrown to the app dev, but will be returned as part of the exception object in the results callback. The Android SDK defines three exception objects and one error object:

·        Buddy Service Exception (parameter / request parsing / handling error)

·        Buddy Service Unknown Error Exception (any unrecognized error returned from a service request)

·        Buddy Service Error (not an exception but a log entry for the failed Buddy request).

The exception handling requirements are summarized in the table below.

Exception source

Examples

Exceptions due to client code errors.

Invalid parameter values or image files that cannot be loaded. The app dev can decide how to take corrective action.

Installing Buddy Android SDK Using Eclipse

This assumes you already have the Android SDK, Eclipse, and the ADT Eclipse plug-in installed. The Android SDK and related development tools can be downloaded from here: http://developer.android.com/sdk/index.html.

1.      Create an Android application

2.      Once created, right-click on your project in Package Explorer and select Properties

3.      Select Java Build Path, Libraries

4.      Select Add External Jars and add the Buddy Android SDK jar and the latest GSON jar library

5.      Copy the Buddy Android SDK jar and GSON library to a folder called libs in the root directory of your project

6.      Start using!

For more details see http://developer.android.com/guide/faq/commontasks.html#addexternallibrary

Buddy Android SDK Samples

The Android SDK includes three sample projects:

·        LocationFinder: shows how to use the Buddy Places API to find places based on your latitude and longitude. This project requires no configuration.

·        ProfilePictureSample: shows how to load, filter and save pictures to Buddy. No configuration is required to run this project.

·        PushNotificationSample: allows Android push notifications to be sent to the device from Buddy. Although the project has been configured to run on the simulator, no push notifications will be sent; the app will simply support registration and unregistration with the Buddy. To use the app on a device you will need to sign it with an appropriate push-compatible certificate. You will also need to register this certificate with your App account at dev.buddy.com.

Keyword Parameters in Android SDK

Many of the Buddy SDK methods support a large number of parameters. Most of these parameters have default values and you don't need to specify them manually. They can be set to null.

Also, nearly all API methods have an overloaded version that only contains the required parameters. Here is an example:

Android (Java)

User.getPlaces().find(1000000, 0.0, 0.0, 10, "Starbucks", 870, null, null); - call with optional parameters

User.getPlaces().find(1000000, 0.0, 0.0, 10, null); - call with required parameters

Callback Types

All callback parameters fall into two generic types:

·        ListResponse

·        Response

Each public method call in the SDK can take a callback type as a parameter. The SDK documentation describes in detail the type assigned to the response callback type. For example, the BuddyClient.Login() callback takes a Response<AuthenticatedUser> object:

BuddyClient client = new BuddyClient("APPNAME", "APPPASS", this.applicationContext());

client.login(token, password, state, new OnCallback<Response<AuthenticatedUser>>()

{

public void OnResponse(Response<AuthenticatedUser> response, Object state) {

                              if (response != null)

                              {

                                             AuthenticatedUser user = response.getResult();

                                             user.getPhotoAlbums().create("My Album", false, null);

                              }

               }

});

The response types contain the result of the call including an exception if the call failed.

Advanced Users

This section is for developers who wish to modify the Buddy SDK source and run the unit tests accompanying the SDK source code.

Installing Buddy Android SDK Source Code Using Eclipse

This assumes you already have the Android SDK, Eclipse, and the ADT Eclipse plug-in installed.

1.      Download the SDK source code from https://github.com/BuddyPlatform/Buddy-Android-SDK.

2.      Open Eclipse and select the source code location in the Select a workspace dialog.

3.      Select the Workbench option.

4.      Right click in the Package Explorer and select Import.

5.      Select General -> Existing Projects into Workspace on the Import dialog.

6.      Browse to the Android directory, in the Select root directory item.

7.      Ensure all projects listed are selected and press Finish. The SDK source code is now loaded.

Run Unit Tests

To run the unit tests within the Buddy SDK Source Code in Eclipse:

1.      Ensure the following SDK code projects are loaded in Eclipse: BuddySDK, BuddySDKTests, and BuddyUnitTestHarness

2.      Ensure a device is connected to the PC, or an Android Virtual Device is running

3.      Select BuddySDKTests, right click and select Run As (or Debug As) -> Android JUnit Test

4.      The unit tests will start running

To run the unit tests from the command line using ANT:

1.      Ensure that ANT has been configured on the PC running the tests – see http://ant.apache.org/manual/install.html for details.

2.      Ensure a device is connected to the PC or an Android Virtual Device is running

3.      Open the command prompt and change directory to the root directory of the Buddy SDK Source code

4.      Run the following commands in this order:

5.      ant clean debug –buildfile Lib/build.xml

6.      ant clean debug -buildfile Tests/BuddyUnitTestHarness/build.xml

7.      ant clean debug install test -buildfile Lib/build.xml

8.      The test output will be shown in the command window

Validation

When running Validation on the code (select all projects, right-click, select Validation), there may be warnings about XML grammar that are benign. To ignore these warnings, go to Preferences from the Window menu, and then navigate to XML -> XML Files -> Validation. In the Validating files section of the preferences panel on the right, choose Ignore in the drop down box for the No grammar specified preference. The file may need to be closed and reopened to make the warnings go away.
